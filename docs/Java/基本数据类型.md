#### Java的基本数据类型

1. 一共有8种数据类型，6种数字类型，一种字符类型，一种布尔类型

* byte类型，是8位数据类型，范围为2^7次方之间,byte 类型用在大型数组中节约空间，主要代替整数，因为 byte 变量占用的空间只有 int 类型的四分之一；

* short数据类型,是16位，有符号的整数类型，范围再2^15次方范围之内,short数据类型，一个short变量是int变量所占空间的二分之一

* Int数据类型，int 数据类型是32位。他的一般默认值就是int类型

* long 表示64位，这种类型主要使用在需要比较大整数的系统上

```java
long a = 10000L
```

* float float 数据类型是单精度、32位、符合IEEE 754标准的浮点数，不能表示精确的值

```Java
float f1 = 23.4f
``` 

* double，精度为双精度，64位，浮点数的默认类型为double类型；

```Java
double d1 = 123.4
```

* boolean,默认值是false，只能取两个值

```Java
boolean b1 = true
```

* char 字符类型，单一16位字符型

```Java
char s1 = "xxx"
```

#### 访问控制修饰符

* default (即默认，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法。

* private  在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）
* public 对所有类可见。使用对象：类、接口、变量、方法
* protected 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）

#### Java的数组

* 声明类型 变量的值
* 数组的长度是固定的，一旦创建超出数组长度则会报错
* 数组动态创建
* 数组静态创建
* 无论是基本数据类型还是原始数据类型，都是存在堆中的
* 数组是相同数据类型的集合

```Java
int [] num; //声明一个数组类型

int num [];//可以声明但是不推荐

int[] num = new  int[10] //可以存放10个int 类型值

int [] num = {1,2,3,4,5.6} //静态初始化
```

#### 类

* 类中只包含了属性和方法 

* static 方法和类初始化时一起实例化

* 值传递和引用传递（Java中都是值传递）

* 基本数据类型都是值传递，对象类型都是引用传递

* 类的构造器（构造方法）

```Java

    public Person(){
        this.name = 'xxx'
    } //和类名相同并且无返回值

//有参构造器,用来初始化值
    public Person(String name){
        this.name = name
    }
}
```

#### 封装

* 高内聚，低耦合
* pravite关键字
* 隐藏变量，必须通过set和get去进行获取和设置

#### 继承

* 继承就是扩展类的属性
* Java都是单继承
* 子类继承了父类就会有父类全部的属性和方法
* 私有属性无法继承，只能通过get/set方法进行获取和赋值
* 所有的对象都继承ObJect类

#### super方法

* 构造器要么调用子类构造器，要么调用父类构造器
* super就是调用父类的构造方法，必须要构造方法的第一个
* this代表谁调用就是代表谁，super代表父类

#### 方法重写

* 需要有继承关系
* 子类重写父类
* 修饰符的范围可以扩大但是不能缩小 pubilc>protector>default>praviate
* 抛出的异常可以缩小但是不能扩大
* 字类重写方法体不能一样

#### 多态

* 同一个方法对不同对象有不同的行为
* 一个对象的实际类型是确定的
* 指向的引用类型的类型是不确定的
* 对象的方法调用，主要是看左边的方法类型
* static方法不能重写，final方法不能重写 praviate方法不能重写

#### instanceof方法类型转换

* 用于引用类型的类型转换
* 满足父子类型的就是true
* 同级的类型不能进行类型转换
* 类型转换，字类转换成父类，可能会丢失方法
* 父类指向子类对象，强制转换，强制转换可能会丢失精度

#### static关键字

* 静态变量存放再方法区中
* 静态变量可以通过类名方法
* 静态方法可以直接通过类调用

##### 抽象类

* 只有类定义，没有内部实现
* 抽象类的继承子类都必须进行类实现
* 抽象方法没有方法实现，只有方法定义
* 抽象类不能new 只能通过子类区实现

#### 接口

* 接口可以多继承，类只能单继承
* 接口这是一种约束，定义一种约束类型
* interface是一个接口定义
* 接口中的方法都是抽象的
* 接口需要要有一个接口实现类
* 实现接口后必须重写接口中的所有方法
* 接口中定义方法
* 方法都是puiblic abstract修饰的
* 接口可以实现多个
* 接口中的方法需要重写
* implements关键词实现接口

#### 内部类的定义

* 再类的内部定义一个类
* 内部类可以访问外部类的私有属性和方法
* 静态内部类
* 匿名内部类(再实例化时不定义名称，不会再内存中储存)
* 局部内部类
